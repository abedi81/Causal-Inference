Fuzzy Regression Discontinuity

This projectexamines the impact of a tutoring program designed for students with low entrance exam scores. The rule of the program is if students scoring below 70 are eligible for tutoring, while those scoring 70 or higher are not. But in practice, things don’t go wll and some eligible students do not participate, and some ineligible students still receive tutoring. This makes the setting a case of fuzzy regression discontinuity, where the cutoff influences the probability of treatment but does not determine it perfectly.

The main question is: does tutoring improve students’ performance on the exit exam, and if so, by how much?

The data are examined for compliance to find out how many students followed the rule, how many eligible students missed out, and how many ineligible students managed to get tutoring anyway. This step highlights the program’s imperfections, motivating the need for an instrumental variable approach.

To estimate the causal effect, the cutoff rule as an instrument for tutoring participation is used. First, a two-stage least squares (2SLS) strategy is implemented manually: the first stage predicts tutoring participation from being below the cutoff, and the second stage estimates how predicted tutoring affects exit exam scores. Then, the same design is implemented formally using an Instrumental Varaible(IV) regression model.

It is fuzzy Regression Discontinuity because in the sharp RD , treatment assignment (tutoring) strictly and clearly determined by running variables (entrance exam). the cut off clearly show who should get the treatment and who should not. But fuzzy RD does not perfectly align with threshold and those one who should not receive the treatment get the treatment and those one who should get the treatment , don’t get that.

We can tell that because there are treated individuals (blue points) below the threshold and control individuals (black points) above the threshold. This shows that treatment status is not perfectly determined by the threshold, showing a fuzzy RD characterastics.

it means 1:because treatment status is not perfectly determined by the threshold, we cannot simply compare the average outcomes above and below the threshold to estimate the treatment effect and we need to approach estimation with the probability of treatment at the threshold and estimate the causal effect for individuals whose treatment is influenced by the threshold.so estimation would be probabilistic rather than deterministic. 2:we also need to pay specific attention to the subpopulation of compliers who are influenced by the threshold. because all the population is not divided cleraly based on cut off. 3:finally to estimate the treatment effect in a fuzzy RD, we need to account for the fact that treatment status is not perfectly determined by the threshold which involving using instrumental variables (IV) to address the fuzziness in treatment assignment.we can use two stage regression model in which in the the first stage we identifies the discontinuity in the probability of treatment at the threshold and then in the second stage we isolates the causal effect of treatment on the outcome (LATE). This approach allows us to estimate the treatment effect for the subpopulation of compliers.

#Centering the running variable and estimate the LATE manually for a +/- 10 point window


# Centering the running variable
exam_data = exam_data %>%
  mutate(
    entrance_exam_centered = entrance_exam - 70,
    below_cutoff = ifelse(entrance_exam_centered < 0, 1, 0)
  )

# data in ±10 point window
exam_data_window = exam_data %>%
  filter(entrance_exam_centered >= -10 & entrance_exam_centered <= 10)

#  First-Stage Regression (Predict Tutoring Program Participation)
first_stage = lm(
  tutoring ~ entrance_exam_centered + below_cutoff,
  data = exam_data_window
)
exam_data_window = exam_data_window %>%
  mutate(tutoring_program_hat = predict(first_stage))

#  Second-Stage Regression (Outcome Regression)
second_stage = lm(
  exit_exam ~ entrance_exam_centered + tutoring_program_hat,
  data = exam_data_window
)


summary(first_stage)

# Esimate the LATE using ivreg() using the same window.


exam_data = exam_data %>%
  mutate(
    entrance_exam_centered = entrance_exam - 70,
    below_cutoff = ifelse(entrance_exam_centered < 0, 1, 0)
  )


exam_data_window = exam_data %>%
  filter(entrance_exam_centered >= -10 & entrance_exam_centered <= 10)

# LATE using Instrumental Variables Regression
iv_model = ivreg(
  exit_exam ~ entrance_exam_centered + tutoring | entrance_exam_centered + below_cutoff,
  data = exam_data_window
)


summary(iv_model)

The result shows that both approaches (2SLS and IV) produce consistent results in which tutoring increases exit exam performance by roughly 9 to 10 points for students near the cutoff.

The key takeaway is that, despite imperfect compliance, the eligibility rule provides a credible source of variation that allows us to measure the causal impact of tutoring. The result is showing the impact of the Local Average Treatment Effect (LATE) which highlight the role of tutoring(treatment) for students whose participation was influenced by being just below the cutoff.

This project shows that tutoring has a clear and significant positive effect on student outcomes, raising exit exam scores by nearly 10 points for borderline students, and it demonstrates how fuzzy regression discontinuity can be used to evaluate real-world programs when rules are not followed perfectly.
